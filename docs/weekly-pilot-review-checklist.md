# Weekly Pilot Review Checklist

Use this every week before exporting the operating review markdown.

## 1) Commitment Pipeline
- [ ] Confirm each pilot commitment has current `status` and `nextStep`.
- [ ] Verify target start dates are still accurate.
- [ ] Mark stalled deals as `on_hold` or `closed_lost` explicitly.
- [ ] Confirm at least one proposal-stage deal has a concrete close action this week.

## 2) Run Quality + Reliability
- [ ] Review success rate and readiness rate from admin metrics.
- [ ] Identify top failure code and assign one owner for fix/mitigation.
- [ ] Confirm median/P95 latency trend is stable versus prior week.

## 3) ROI + Customer Proof
- [ ] Update baseline vs current KPI deltas (denial rate, days-to-auth, hours saved, revenue).
- [ ] Export KPI snapshot and case-study draft markdown.
- [ ] Capture one operator quote tied to measurable impact.

## 4) Decisions + Next 7 Days
- [ ] Decide: scale lane, hold, or stabilize.
- [ ] Lock 3 commitments for next 7 days (one demo, one close action, one reliability fix).
- [ ] Assign owner and due date for each commitment.
