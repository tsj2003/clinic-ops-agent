# Targeted Demo Day Playbook (Day 6)

Goal: run focused wedge demos and convert to paid pilot conversations.

## Audience
- Spine/pain clinic ops leads
- Prior-auth managers
- Revenue cycle leaders

## Demo Flow (Strict)
1. Pain statement (manual payer research bottleneck)
2. Live run proof (policy + contact workflow states)
3. Readiness verdict + operator handoff
4. KPI framing (denial-rate delta, days-to-auth delta, hours saved, recovered revenue)
5. Paid pilot ask (14-day, one-lane, success criteria locked)

## Talk Track
Use `docs/final-demo-script.md`.

## Objection Handling

### "We already have prior-auth software"
"We don’t replace your full stack. We remove manual readiness/routing work before submission."

### "This is too much workflow change"
"Pilot starts with one lane and one user. No broad rollout needed."

### "How do we evaluate success"
"Weekly KPI reviews with locked success criteria at kickoff."

## Same-Day Patch Loop
After each demo:
- Capture objection
- Classify as product / messaging / security / pricing
- Patch docs or UI same day
- Update call script before next demo
